⚙️ COMPONENT BREAKDOWN
🧠 1. SERVER (the brain)
🔹 Responsibilities
Accept WebSocket connections
Identify agents
Store active agents
Route messages
Send commands
🔌 WebSocket Endpoint
ws://yourserver/ws
🧠 Agent Registry (IMPORTANT)

In memory for now:

type Agent struct {
    ID        string
    Conn      *websocket.Conn
    LastSeen  time.Time
}

Store in:

map[string]*Agent
📩 Message Format (DON’T CHANGE RANDOMLY)

Keep it clean JSON:

{
  "type": "register",
  "agent_id": "abc123"
}
Other message types:
{ "type": "heartbeat" }

{ "type": "command", "command": "whoami" }

{ "type": "result", "output": "root" }
🔄 Flow (Server side)
Agent connects
Sends register
Server stores agent
Controller requests agent list
Controller sends command
Server forwards to agent
Agent responds → server → controller
🤖 2. AGENT (your little soldier)
🔹 Responsibilities
Connect to server
Register itself
Stay alive (heartbeat)
Execute commands
Send results
🔌 Connection Logic
ws://server/ws

Auto-reconnect:
👉 if connection drops → retry every 5s

🧠 Register on connect
{
  "type": "register",
  "agent_id": "unique-machine-id"
}

👉 Generate ID:

hostname + random hash
OR
persistent UUID file
❤️ Heartbeat

Every 10–20 seconds:

{ "type": "heartbeat" }
⚙️ Command Execution

When receiving:

{ "type": "command", "command": "whoami" }

Run:

Windows → cmd /C
Linux → /bin/sh -c

Return:

{
  "type": "result",
  "output": "femonla"
}

⚠️ Don’t overcomplicate:
No sandbox yet, just raw execution (you’re testing your own machines anyway).

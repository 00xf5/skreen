The platform

Built differently from the start

01
Persistent agents, not sessions
The Skreen agent installs silently and survives reboots. It registers itself to your controller on startup and stays ready — no user action required on the remote end.
02
Your infrastructure, your rules
Deploy the Skreen server on any machine you own. No data leaves your network unless you decide otherwise. Compliance, data residency, and audit requirements stay within your control.
03
Real-time screen access
WebRTC-based screen streaming delivers low-latency video of the remote desktop. No plugins, no viewer software — your operator sees the endpoint directly in their browser.
04
Full command execution
Run shell commands, PowerShell scripts, or system queries against any connected agent from an in-browser terminal. Output is streamed back immediately over a persistent WebSocket connection.
How it works

From install to connected in under two minutes

01
Deploy the Skreen server
Run the Skreen server binary or Docker container on any machine with a public IP or domain. It listens for agent connections and serves the operator controller. A Caddy reverse proxy configuration is included for HTTPS termination out of the box.
02
Build and distribute the agent
From the operator dashboard, generate a pre-configured agent binary baked with your server's address and an invite code. Send the file to your end user or deploy it via your existing software management tooling.
03
The agent registers automatically
Once launched, the agent connects back to the controller over an encrypted WebSocket connection. It registers its hostname, operating system, and hardware profile, then enters a permanent listen state. If connectivity drops, it reconnects on its own.
04
Operate from the controller
Sign in to the Skreen web controller. Select any online agent. Open a live terminal, view the remote screen, transfer files, or inspect running processes — all without leaving the browser tab.
Capabilities

Everything an operator needs. Nothing extra.

Live screen sharing
Real-time WebRTC video stream of the remote desktop. Low latency, no plugin required on either side.
Remote terminal
In-browser shell with live output streaming. Supports PowerShell, cmd, and bash depending on the target OS.
File transfer
Browse the remote filesystem and upload or download files directly through the controller interface.
Process management
View all running processes on the remote endpoint. Kill or inspect any process from the operator dashboard.
Invite links
Generate a time-limited access code and share a join URL with an end user. They download and run the agent with a single click — no technical knowledge required.
Audit log
Every connection, command, and file operation is logged with a timestamp. Exportable for compliance and review.
Persistence on reboot
The agent optionally installs as a Windows service or systemd unit, reconnecting automatically after any restart.
TURN relay support
Integrated Coturn configuration for WebRTC traversal through strict firewalls and symmetric NAT environments.
Operator authentication
Sign-in with email and password or Google via Firebase Authentication. Each operator session is fully isolated.

On self-hosting

Why this matters more than you might think

Most remote access tools on the market operate as cloud brokers. Your connection travels through their relay network, your session metadata is stored on their servers, and your continued access depends on their business staying solvent and their pricing remaining acceptable.

Skreen takes a different position. The server is software you run. The data is yours. If you want to take the installation offline, modify how it works, or integrate it with your existing tooling, nothing stops you. The binary is yours.

This is not a philosophical stance. It is a practical one. Teams managing sensitive infrastructure should not have to negotiate with a SaaS vendor's terms of service to do their jobs. Skreen is infrastructure — it should behave like infrastructure.
